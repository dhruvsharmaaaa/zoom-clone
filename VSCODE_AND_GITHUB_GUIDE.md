# VS Code + GitHub: Step-by-Step (for first-timers)

Follow this in order. Every step assumes you've never used VS Code's
terminal before.

## Step 0 — Get the project onto your laptop
1. Download `zoom-clone.zip` (linked in the chat).
2. Extract/unzip it somewhere easy to find, e.g. `Desktop/zoom-clone`.

## Step 1 — Open it in VS Code
1. Open VS Code.
2. `File → Open Folder...` → select the extracted `zoom-clone` folder → Open.
3. You'll see a file tree on the left with `backend/`, `frontend/`, and the
   `README.md` etc.

## Step 2 — Open a terminal inside VS Code
- Menu: `Terminal → New Terminal` (or shortcut `` Ctrl+` `` on Windows/Linux,
  `` Cmd+` `` on Mac).
- A terminal panel opens at the bottom, already pointed at your project
  folder. Every command below gets typed here.

## Step 3 — Install two VS Code extensions (optional but helpful)
Click the Extensions icon in the left sidebar (looks like 4 squares) and
install:
- **Python** (by Microsoft) — gives you syntax highlighting/IntelliSense for `backend/`.
- **ES7+ React/Redux snippets** or just the built-in JS support is fine for `frontend/`.

## Step 4 — Run the backend
In the terminal:
```bash
cd backend
python -m venv venv
```
Activate the virtual environment:
- **Mac/Linux**: `source venv/bin/activate`
- **Windows**: `venv\Scripts\activate`

You'll see `(venv)` appear at the start of your terminal line — that means
it worked. Then:
```bash
pip install -r requirements.txt
python -m app.seed
uvicorn app.main:app --reload --port 8000
```
Leave this terminal running. Open `http://localhost:8000/docs` in a browser
— you should see FastAPI's interactive API documentation. That confirms
the backend works.

## Step 5 — Run the frontend (in a SECOND terminal)
Don't close the backend terminal. Open a new one:
`Terminal → New Terminal` again (or click the `+` icon in the terminal panel).
```bash
cd frontend
npm install
npm run dev
```
Open `http://localhost:3000` in your browser — you should see the Zoom-style
dashboard.

## Step 6 — Test it actually works
- Click **New Meeting**, type a name, start it.
- Copy the URL bar's meeting link.
- Open a **second browser tab** (or a different browser) with that link,
  enter a different name, join.
- Allow camera/mic access when your browser asks. You should see both
  video feeds.

## Step 7 — Push to your own GitHub (so you can submit the repo link)

1. **Create a GitHub account** if you don't have one: https://github.com/join
2. **Create a new empty repository**:
   - Go to https://github.com/new
   - Name it e.g. `zoom-clone`
   - Keep it **Public** (assignment requires this)
   - Do NOT check "Add a README" (we already have one) — leave it empty.
   - Click **Create repository**. GitHub will show you a page with commands
     — you don't need to copy those, use the ones below instead.

3. **Back in VS Code's terminal**, from the project ROOT folder (not
   `backend/` or `frontend/` — type `cd ..` first if you're still inside
   one of those):
```bash
git init
git add .
git commit -m "Initial commit: Zoom clone SDE assignment"
git branch -M main
git remote add origin https://github.com/<your-username>/zoom-clone.git
git push -u origin main
```
Replace `<your-username>` with your actual GitHub username. When it asks
for credentials, use a GitHub **Personal Access Token** as the password
(GitHub stopped accepting plain passwords for git operations) — VS Code
will usually pop up a browser window to let you sign in instead, which is
easier; just click "Authorize" if prompted.

4. Refresh your GitHub repo page in the browser — you should see all your
   files there. Copy that repository URL for your submission.

## Common issues

- **`git: command not found`**: install Git from https://git-scm.com/downloads,
  restart VS Code, try again.
- **`python: command not found`**: try `python3` instead of `python` on
  Mac/Linux.
- **Port already in use** (8000 or 3000): something else is using that
  port. Stop it, or change `--port 8000` to another number (and update
  `frontend/.env.local`'s `NEXT_PUBLIC_API_URL` to match).
- **Pushed but `node_modules` / `venv` are missing from GitHub**: that's
  correct and intentional — `.gitignore` deliberately excludes them (they're
  huge and regenerated by `npm install` / `pip install`). Anyone cloning
  your repo just re-runs the setup steps above.
